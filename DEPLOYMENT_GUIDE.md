# Deployment Guide for Jay's Phone Repair App

This guide outlines the steps to deploy the mobile app for production so users can download and install it from the web browser.

## Prerequisites

1. Expo Account (jaymuthui)
2. Google Play Developer Account (for Android)
3. Apple Developer Account (for iOS)
4. Proper app icons and splash screens
5. App store descriptions and screenshots

## 1. Configure Expo for Production Builds

### Update app.json with production settings:

```json
{
  "expo": {
    "name": "Jay's Phone Repair",
    "slug": "jays-phone-repair",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "userInterfaceStyle": "automatic",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#3b82f6"
    },
    "ios": {
      "supportsTablet": true,
      "bundleIdentifier": "com.jaysphonerepair.app",
      "buildNumber": "1.0.0",
      "infoPlist": {
        "NSCameraUsageDescription": "This app needs camera access to take photos of repairs.",
        "NSPhotoLibraryUsageDescription": "This app needs photo library access to select repair images."
      }
    },
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#3b82f6"
      },
      "package": "com.jaysphonerepair.app",
      "versionCode": 1,
      "permissions": [
        "CAMERA",
        "READ_EXTERNAL_STORAGE",
        "WRITE_EXTERNAL_STORAGE"
      ]
    },
    "web": {
      "favicon": "./assets/favicon.png"
    }
  }
}
```

## 2. Build Standalone Apps with EAS

### Install EAS CLI:
```bash
npm install -g eas-cli
```

### Log in to Expo:
```bash
eas login
```

### Build for Android:
```bash
eas build -p android --profile production
```

### Build for iOS:
```bash
eas build -p ios --profile production
```

## 3. Web Deployment

### Build the web version:
```bash
cd mobile-app
expo export:web
```

### Deploy to hosting platform (Vercel/Netlify/GitHub Pages):
1. Copy the `web-build` folder contents
2. Deploy to your preferred hosting platform

## 4. Update Homepage Download Links

Update the download links in `src/components/homepage/DownloadAppSection.tsx`:

1. Android APK link - Point to the actual APK file location
2. iOS App Store link - Point to the actual App Store listing
3. Expo Go link - Point to your Expo project

## 5. App Store Submission

### For Google Play Store:
1. Create application listing
2. Upload the AAB file generated by EAS
3. Fill in app details, screenshots, and descriptions
4. Submit for review

### For Apple App Store:
1. Create application listing in App Store Connect
2. Upload the IPA file generated by EAS
3. Fill in app details, screenshots, and descriptions
4. Submit for review

## 6. Testing Installation Process

Before going live, test the complete installation process:

1. Visit the homepage from a mobile device
2. Click the Android/iOS download button
3. Verify the app downloads and installs correctly
4. Test all major app features

## 7. Monitoring and Updates

1. Monitor app performance and crashes
2. Collect user feedback
3. Plan regular updates with new features
4. Use Expo OTA updates for minor bug fixes

## Troubleshooting

### Common Issues:

1. **Download fails**: Check file paths and hosting configuration
2. **Installation blocked**: Ensure proper app signing and permissions
3. **App crashes on startup**: Check logs and ensure all dependencies are properly configured
